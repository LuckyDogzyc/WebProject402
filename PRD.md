# PRD: 402 虚拟生活社区系统

## 1. Executive Summary

**项目名称**：402 虚拟生活社区系统  
**版本**：v1.0  
**创建日期**：2026-04-20  
**产品类型**：游戏化虚拟社交网页平台  
**核心目标**：为 6 人固定团队构建一个融合虚拟角色、互动场景与轻量社交功能的个性化网页系统，打造属于 402 小队的"数字家园"。

---

## 2. Mission

为 402 小队成员提供一个沉浸式、情感化的虚拟生活空间，通过游戏化互动与个性化表达，增强团队凝聚力与归属感。

---

## 3. Target Users

### 核心用户
- **固定用户群**：6 名 402 小队成员（无动态注册/注销）
- **成员 ID**：初、博、昊、千、东、竹（可随时修改）
- **用户画像**：
  - 年龄：20-30 岁
  - 特征：熟悉互联网文化，喜爱游戏化体验
  - 需求：情感表达、轻度娱乐、团队互动

### 用户角色
| 角色 | 数量 | 说明 |
|------|------|------|
| 超级管理员 | 1 | 系统初始搭建与运维 |
| 普通成员 | 5 | 日常互动用户 |

## 4. MVP Scope

### 核心功能模块

#### 4.1 音乐播放系统
- **功能**：内置 MP3 播放器，首曲固定为《402》，支持循环播放
- **歌单管理**：支持自定义歌单，提供歌单选择功能
- **技术实现**：前端播放器（HTML5 Audio + 自定义 UI）

#### 4.2 虚拟角色系统（基于 Phaser 3）
- **角色创建**：每位用户可创建并定制个人虚拟形象
- **定制维度**：
  - 肖像（像素风头像）
  - 性格特质
  - 能力定位（主C、辅助、近战等）
  - 皮肤系统（节日皮肤 + "绿头鱼"套装）
- **动画系统**：
  - 待机动画（4 帧）
  - 行走动画（8 帧）
  - 工作动画（4 帧）
  - 醉酒状态（2 帧）
- **换装系统**：支持角色服装与风格切换
- **技术实现**：Phaser 3 游戏引擎 + 像素风精灵图

#### 4.3 风格化皮肤系统
- **节日皮肤**：根据节日氛围动态切换角色皮肤
- **风格统一**：全局采用**像素风**（Pixel Art）
- **初始皮肤**："绿头鱼"套装

#### 4.4 虚拟生活场景（基于 Phaser 3）
- **固定空间**：构建虚拟房间/生活场景（2D 俯视图）
- **场景交互**：
  - 点击角色显示交互菜单
  - 拖拽物品进行互动
  - 场景物品可放置/移除
- **自动行为**：角色自动进行起床、洗漱、上班等行为（无需用户操作）
- **物理引擎**：基于 Phaser Arcade Physics 的碰撞检测
- **生态圈**：形成自运行的"小生态圈"
- **技术实现**：Phaser 3 Scene + Arcade Physics

#### 4.5 互动行为系统
- **调酒互动**：用户可与虚拟角色进行调酒互动
- **灌醉机制**：角色被灌醉后几小时内无法行动
- **备忘录**：支持设置个人备忘录
- **虚拟黑板**：公共留言板功能

#### 4.6 角色身份认领
- **生活化角色**：6 人团队可认领角色（厨师、氛围担当、打手等）
- **随时切换**：支持随时切换"今日状态"

#### 4.7 树洞匿名系统
- **匿名情绪**：用户可输入负面情绪
- **画像绘制**：他人根据描述绘制对应人物画像
- **身份保护**：不暴露发布者身份

#### 4.8 旅行地图与照片墙
- **地图记录**：记录已到访地点（苏州、呼和浩特等）
- **照片墙**：点击地点可查看该地照片墙
- **外存方案**：照片通过**腾讯云 COS** 存储，本地仅保留索引

#### 4.9 内部聊天室
- **私密交流**：支持 5-6 人私密聊天
- **技术方案**：使用开源聊天方案（如 WebSocket + 自研轻量协议）

### 暂不包含功能
- 用户注册/注销（固定 6 人）
- 密码系统（无认证需求）
- 支付/商业化功能

## 5. User Stories

### 音乐播放
> 作为 402 成员，我希望在进入网页时能自动听到《402》这首歌，这样我能快速进入状态。

### 虚拟角色
> 作为 402 成员，我希望创建一个属于自己的虚拟形象，包括外貌、性格和能力定位，这样我能在虚拟空间中展现自我。

### 身份切换
> 作为 402 成员，我希望今天可以当"厨师"，明天当"打手"，这样能体验不同的角色扮演乐趣。

### 树洞情绪
> 作为 402 成员，当我心情不好时，我希望能在树洞匿名倾诉，并由其他人画一幅画来安慰我。

### 旅行回忆
> 作为 402 成员，我希望能在地图上标记我们一起去过的地方，点击就能看到当时拍的照片。

## 6. Core Architecture & Patterns

### 系统架构
```
┌─────────────────────────────────────────────────────┐
│                   前端层 (Browser)                    │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐ │
│  │ 音乐播放 │  │ 虚拟场景 │  │ 聊天室  │  │ 地图照片 │ │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘ │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP/WebSocket
┌──────────────────────┴──────────────────────────────┐
│                  后端层 (Server)                      │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐ │
│  │ 用户识别 │  │ 数据存储 │  │ 聊天服务 │  │ 图片代理 │ │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘ │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────┴──────────────────────────────┐
│                  存储层 (Storage)                     │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐              │
│  │ SQLite  │  │ 图床CDN │  │ 本地文件 │              │
│  └─────────┘  └─────────┘  └─────────┘              │
└─────────────────────────────────────────────────────┘
```

### 核心模式
- **前端渲染**：所有 UI 在浏览器中动态生成
- **轻量后端**：Python Flask/FastAPI 提供 API 与 WebSocket
- **数据持久化**：SQLite + JSON 文件存储
- **图片外存**：图床 CDN（阿里云 OSS / 腾讯云 COS）

## 10. API Specification

### RESTful API

#### 用户识别
```
POST /api/identify
Request: { "mac": "xx:xx:xx:xx:xx:xx" }
Response: { "userId": "user_01", "userName": "赵寅初" }
```

#### 成员信息
```
GET /api/members
Response: [{ "id": "user_01", "name": "赵寅初", "role": "厨师" }]
```

#### 角色状态
```
GET /api/characters
Response: [{ "userId": "user_01", "state": "sober", "action": "working" }]

POST /api/characters/action
Request: { "userId": "user_01", "action": "make_drink" }
```

#### 树洞情绪
```
GET /api/treehole
Response: [{ "id": 1, "content": "今天好难过", "image": "..." }]

POST /api/treehole
Request: { "content": "今天好难过", "anonymous": true }
```

#### 地图照片
```
GET /api/locations
Response: [{ "id": "suzhou", "name": "苏州", "photos": ["url1", "url2"] }]
```

### WebSocket 事件

#### 聊天室
```javascript
// 发送消息
socket.emit('chat', { userId: 'user_01', content: '大家好', timestamp: Date.now() })

// 接收消息
socket.on('chat', (msg) => { console.log(msg) })
```

#### 角色状态同步
```javascript
// 状态变更
socket.emit('character_state', { userId: 'user_01', state: 'drunk', duration: 3600000 })
```

## 14. Risks & Mitigations

### 技术风险
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| WebSocket 兼容性 | 聊天功能异常 | 提供轮询降级方案 |
| 图床限流 | 图片加载失败 | 本地缓存 + 多图床备份 |
| 浏览器存储限制 | Cookie/LocalStorage 被清空 | 提供 MAC 地址识别备用方案 |

### 业务风险
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 6 人固定限制 | 无法扩展 | 设计预留扩展接口 |
| 内容审核缺失 | 树洞内容不当 | 增加简单过滤/举报机制 |

### 运维风险
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 服务器宕机 | 服务不可用 | 定期备份 + 监控告警 |
| 图床费用超支 | 运营成本增加 | 设置用量上限 + 压缩优化 |
